# Архитектура стенда TT-RSS

## Назначение сервисов

### `db`

Сервис `db` запускает PostgreSQL и хранит данные TT-RSS:

- учетные записи пользователей
- настройки приложения
- список RSS-источников
- загруженные и обработанные записи

Данные базы сохраняются в именованном томе `tt_rss_db_data`, поэтому не теряются при обычной остановке контейнеров.

### `tt-rss`

Сервис `tt-rss` запускает веб-приложение Tiny Tiny RSS.  
Он:

- подключается к PostgreSQL
- инициализирует приложение при первом запуске
- предоставляет веб-интерфейс на указанном порту

Данные приложения сохраняются в именованном томе `tt_rss_app_data`.

## Как сервисы взаимодействуют

1. Docker Compose поднимает контейнер `db`.
2. PostgreSQL проходит проверку готовности.
3. После этого запускается контейнер `tt-rss`.
4. `tt-rss` подключается к базе по имени сервиса `db` внутри docker-сети.
5. Пользователь открывает веб-интерфейс через браузер на порту, указанном в `.env`.

## Схема взаимодействия

```text
Браузер --> tt-rss --> PostgreSQL
           :80         :5432
```
