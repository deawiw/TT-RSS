# Описание проекта: Тематический отбор новостных материалов по кибермошенничеству

## 1. Проблема

В современном мире количество мошеннических действий, направленных на граждан и организации, стремительно растёт. Злоумышленники постоянно разрабатывают новые схемы обмана: фишинговые письма, поддельные ссылки, звонки с подменой голоса (voice spoofing), фейковые сайты банков и государственных услуг, социальная инженерия. Информация о таких угрозах появляется в различных открытых источниках: новостных порталах, блогах по информационной безопасности, официальных предупреждениях, форумах.

Основные проблемы, с которыми сталкиваются потенциальные жертвы:

- **Информационная перегрузка** – огромный поток новостей, в котором сложно выделить действительно важные сообщения о новых видах мошенничества.
- **Неосведомлённость** – многие люди не знают о существующих угрозах или узнают о них слишком поздно.
- **Отсутствие систематизации** – информация разрознена, нет единого инструмента для её сбора, категоризации и оперативного просмотра.
- **Затраты времени** – поиск актуальных предупреждений о мошеннических схемах требует значительных временных ресурсов.

## 2. Цель проекта

Разработать автоматизированную систему сбора, тематической категоризации и визуализации новостных материалов, связанных с мошенничеством и киберугрозами, на базе новостного агрегатора TT-RSS. Система должна предоставлять пользователям удобный интерфейс для оперативного получения структурированной и актуальной информации о новых мошеннических схемах, фишинговых атаках, подменных звонках и других угрозах.

## 3. Основные задачи

1. **Развёртывание инфраструктуры** – установка и настройка TT-RSS в контейнерах Docker для централизованного сбора новостных лент.
2. **Сбор данных** – разработка модуля ETL (R-пакет) для выгрузки новостей через API TT-RSS, очистки, нормализации и сохранения в структурированном виде (датафрейм,PostqreSQL).
3. **Категоризация** – создание модуля машинного обучения (тематическое моделирование, классификация) для автоматического распределения новостей по темам, включая выделение материалов о мошенничестве, фишинге, подмене голоса и т.д.
4. **Визуализация** – разработка интерактивного веб-приложения на Shiny, позволяющего пользователям просматривать новости с фильтрацией по теме, дате, источнику, а также строить графики и отслеживать тренды.
5. **Интеграция с AI** – реализация MCP-сервера (Model Context Protocol) для предоставления доступа к данным AI-агентам и чат-ботам.
6. **Контейнеризация и автоматизация** – упаковка всех компонентов в Docker-образы и организация запуска через docker-compose для простоты развёртывания.

## 4. Целевая аудитория

- **Обычные пользователи** – люди, желающие защитить себя и своих близких от мошенников, быстро узнавать о новых схемах обмана и способах защиты.
- **Организации** – компании, которые хотят информировать своих сотрудников о текущих фишинговых кампаниях и методах социальной инженерии.

## 6. Бизнес-логика и ценность

**Основная бизнес-ценность** – сокращение времени, которое пользователи тратят на поиск и анализ информации о мошеннических угрозах. Система автоматически:

- Собирает новости из множества источников в одном месте.
- Фильтрует и выделяет именно те материалы, которые касаются мошенничества и фишинга.
- Позволяет быстро просматривать заголовки, краткое содержание, переходить к полной статье.
- Предоставляет аналитику: динамика упоминаний, наиболее активные источники, популярные темы.

Это помогает пользователям оставаться в курсе новых угроз, принимать превентивные меры и снижать риск стать жертвой мошенников. Для организаций система служит инструментом повышения осведомлённости сотрудников, что уменьшает вероятность успешных атак через социальную инженерию.

## 7. Техническая архитектура (кратко)

Проект строится на основе микросервисной архитектуры с использованием контейнеризации:

- **TT-RSS** – централизованный агрегатор RSS-лент, развёрнутый в Docker.
- **Модуль ETL** – R-пакет, который через API TT-RSS выгружает данные, очищает и сохраняет в DuckDB (локальное аналитическое хранилище).
- **Модуль ML** – R-пакет с функциями тематического моделирования и классификации на основе текстов новостей (используются библиотеки `quanteda`, `text`, `tidymodels`).
- **Shiny-приложение** – интерактивный дашборд для просмотра и анализа новостей.
- **MCP-сервер** – реализован на R, предоставляет доступ к данным и категориям через протокол Model Context Protocol для интеграции с AI-агентами.
- **Docker Compose** – оркестрация всех контейнеров (TT-RSS, база данных, Shiny, MCP-сервер).

## 8. Ожидаемые результаты

1. Работоспособный R-пакет с функциями сбора, обработки, категоризации новостей.
2. Интерактивное веб-приложение на Shiny, доступное через браузер.
3. Документация по развёртыванию (docker-compose) и использованию.
4. MCP-сервер для интеграции с внешними AI-системами.
5. Примеры собранных данных и результаты классификации по теме мошенничества.

Проект позволяет быстро и эффективно отслеживать информационное поле в области кибермошенничества, предоставляя пользователям структурированный и актуальный контент.
