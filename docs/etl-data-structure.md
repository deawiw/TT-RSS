# Модуль ETL: структура данных

## 1. Назначение модуля ETL

Модуль ETL (Extract, Transform, Load) отвечает за:
- **Извлечение (Extract)** сырых новостей из базы данных TT-RSS через API.
- **Трансформацию (Transform)** – очистка HTML-сущностей (например, `&#1040;` → `А`), удаление дубликатов, приведение дат к единому формату, выделение ключевых полей.
- **Загрузку (Load)** структурированных данных в аналитическое хранилище (DuckDB/PostgreSQL) для последующей работы ML-модуля и Shiny-приложения.

## 2. Входные данные (Raw Data из TT-RSS)

На вход ETL получает от TT-RSS записи в формате, близком к CSV/табличному, со следующими основными полями:

- `article_id` – идентификатор в системе TT-RSS
- `feed_title` – название RSS-ленты/источника
- `title` – заголовок новости
- `content` – HTML-текст новости (может содержать сущности)
- `published_at` – дата и время публикации (строка)
- `link` – URL оригинальной статьи
- `tags` – теги новости (например, "происшествия")
- `loaded_at` – время загрузки записи в TT-RSS


## 3. Выходные данные (После ETL)

На выходе ETL формирует структурированный набор данных (таблицу/датафрейм) со следующими полями, подготовленными для анализа и ML:

| Поле в выходных данных | Тип данных  |                  Описание                 |                          Примечание по трансформации                    |

|       `news_id`        | integer (PK)| Уникальный числовой идентификатор новости |                       Прямое отображение из `article_id`                |
|       `source`         |    string   |         Название источника новости        |                          Отображение из `feed_title`                    |
|       `title`          |    string   |             Заголовок новости             |                           Очищен от HTML-сущностей                      |
|     `content_text`     |    string   |               Текст новости               | Из поля `content`: удалены теги, декодированы сущности (`&#1040;` → `А`)|
|     `published_at`     |   datetime  |        Дата и время публикации (в UTC)    |              Из `published_at`, преобразовано в POSIXct/datetime        |
|        `url`           |    string   |              Ссылка на оригинал           |                           Прямое отображение из `link`                  |
|     `topic_tags`       |    string   |             Исходные теги из RSS          |                                Из поля `tags`                           |
|     `extracted_at`     |   datetime  |  Время загрузки в аналитическое хранилище |              Устанавливается ETL (`loaded_at` или текущее время)        |

## 4. Пример данных (до и после ETL)

**Сырая строка (вход):**
- `title`: `В Ливии заявили, что "Арктик метагаз" дрейфует`
- `content`: `&amp;quot;&#1040;&#1088;&#1082;&#1090;&#1080;&#1082; ...`

**После ETL (выход):**
- `title`: `В Ливии заявили, что "Арктик метагаз" дрейфует`
- `content_text`: `"Арктик Метагаз" сорвался с буксира из-за неблагоприятных погодных условий`

## 5. Логика фильтрации (для целей проекта)

Так как нас интересует **кибермошенничество**, ETL может сразу применять легковесную фильтрацию:

- **Стадия 1 (Сохранение всего):** Пока сохраняем все новости, так как мошенничество может быть в неочевидных разделах.
- **Стадия 2 (Разметка):** Добавляем служебное поле `is_fraud_related` (boolean), которое будет заполняться ML-модулем (True, если новость о фишинге, подмене голоса, социнженерии).
